Getting Started
Open the shared copy
Always open DaySchedule from the shared team copy, not from Downloads, Desktop, email, or a ZIP. The shared copy is the one everyone should use.
Connect once per browser
On first launch, click Connect Shared Folder and choose the folder where schedule files live. In most setups, that folder is app/data.
Create a schedule
From the library, click + New Schedule, give it a name, and start adding days and events. New schedules open in edit mode automatically.
Edit events
Open the schedule, click Edit This Schedule if needed, then click any event on the schedule to open its properties in the inspector panel on the right.
Click the header
Click the schedule header (title/logo area) to open Settings — change the title, logo, contact info, and audience groups.
Saving & Shared Editing
Auto-save
Every change is automatically saved 2 seconds after you stop editing. Watch for Saved in the toolbar. You can also press Ctrl+S to save immediately.
One editor at a time
Recommended team workflow: one person edits a schedule at a time. Use Edit This Schedule to claim editing, then wait for Saved before handing it off.
Take over only when needed
If the current editor is unavailable and the team needs immediate access, a lead can use Take Over. That action should only be used after confirming the handoff or that the current editor is no longer working.
Avoid personal copies
If you edit a downloaded or personal copy, your changes stay there. Other people will keep working from the shared copy, and the two versions can drift apart.
Before handoff
If another person needs to edit after you, return to the library only after you see Saved. For major changes, save a named version first.
Versions
Use the Versions button to save named snapshots (e.g., “Draft for Review”). You can always restore a previous version — the current state is auto-backed up first.

FAQ
Where should I open DaySchedule from?
Open it from the shared team copy every time. If your team reaches it through SharePoint or Teams, use that shared copy on your computer instead of downloading a separate copy.
What folder do I pick?
Choose the folder where the schedule JSON files live. In most DaySchedule copies, that is app/data. If your team packaged it differently, choose the folder that already contains the saved schedules.
Can multiple people edit the same schedule at once?
The safest workflow is no. Have one person edit at a time per schedule, then hand off after the save indicator shows Saved.
What if the last editor stepped away?
Edit access uses a lease. If the last editor stops refreshing it, the lock expires automatically and the next person can claim edit access. If the team lead needs in sooner, they can use Take Over after confirming the current editor is unavailable.
What if I do not see someone else’s changes?
Confirm you opened the shared copy, not a personal one. Then return to the library and reopen the schedule so DaySchedule reloads the latest file.
What browser should I use?
Use Chrome or Edge for the full auto-save experience. Other browsers can still open the app, but saving falls back to manual Save to File.
Tips
Duplicate a schedule
Right-click a schedule in the library to duplicate it. Great for starting next month from this month’s template.
Audience groups
Main groups get full-width bands on the schedule. Limited groups appear as concurrent side notes when they overlap with main events.
