Getting Started
Create a schedule
From the library, tap + New Schedule, give it a name, and start adding days and events.
Edit events
Click any event on the schedule to open its properties in the inspector panel on the right.
Click the header
Click the schedule header (title/logo area) to open Settings — change the title, logo, contact info, and audience groups.
Saving & Teams
Auto-save
Every change is automatically saved 2 seconds after you stop editing. Watch for Saved in the toolbar. You can also press Ctrl+S to save immediately.
Teams setup (important)
You must Sync the Teams folder — not Download it. In your Teams channel, click Files → Sync. This creates a live folder on your computer (usually under OneDrive). Open index.html from there. When the app asks for the data folder, pick it from the same synced location. Changes will automatically sync to everyone on the team.
What happens if I use a downloaded copy?
Your changes stay on your computer only. Nobody else will see them, and you won't see anyone else's changes. If someone else is working from the synced version, your work could be lost. Always use the synced folder.
Versions
Use the Versions button to save named snapshots (e.g., “Draft for Review”). You can always restore a previous version — the current state is auto-backed up first.
Keyboard Shortcuts
Ctrl + SSave now
Ctrl + PPrint all days
Ctrl + ZUndo
Ctrl + Shift + ZRedo
EscClose panels
Tips
Duplicate a schedule
Right-click a schedule in the library to duplicate it. Great for starting next month from this month’s template.
Audience groups
Main groups get full-width bands on the schedule. Limited groups appear as concurrent side notes when they overlap with main events.
Use Chrome or Edge
Auto-save only works in Chrome and Edge. Other browsers fall back to manual download.
Using locally (no Teams)
Everything works without Teams. Just open index.html from wherever you saved DaySchedule and pick the data folder when prompted. The Teams sync steps only apply if you need shared access with others.

Confirm: Is this the synced folder?

The folder you just picked must be inside your Teams-synced location (OneDrive). If you're working from a downloaded copy, your changes won't be shared with the team.
