Schedule Builder

Start Here
0. Use your real name
The first time you edit, DaySchedule asks for your name. Use the real name your team will recognize, because saves, versions, and takeovers are tagged with it.
1. Open the shared copy
Always open DaySchedule from the shared team copy, not from Downloads, Desktop, email, or a ZIP.
2. Connect once
On first launch, click Connect Shared Folder and choose the folder where the saved schedules live. In most setups, that folder is app/data.
3. Edit the schedule
Open a schedule, click Edit if it opens read-only, then use Quick Edit like a worksheet. Click a row to edit the extra fields below, or double-click a row to open full event details.
4. Let Audience place the event
In Quick Edit and Event Details, Audience means the group or section this event belongs to. Primary audiences go to the main track automatically. Supporting or unassigned events stay in the side track unless you turn on Main Track.
5. Use Specific People sparingly
Specific People is only for named exceptions inside the selected audience. In Quick Edit, those extra fields stay together in the selected-row editor at the bottom instead of repeating on every row.
Saving & Shared Editing
Auto-save
Every change is automatically saved 2 seconds after you stop editing. Watch for Saved in the toolbar. You can also press Ctrl+S to save immediately.
One editor at a time
Recommended team workflow: one person edits a schedule at a time. Click Edit, make your changes, wait for Saved, then click Done Editing before handing it off.
Take over only when needed
If the current editor is unavailable and the team needs immediate access, a lead can use Take Over. That action should only be used after confirming the handoff or that the current editor is no longer working.
Avoid personal copies
If you edit a downloaded or personal copy, your changes stay there. Other people will keep working from the shared copy, and the two versions can drift apart.
Customize the schedule
Use Customize to change the title, unit logo, layout, colors, audiences, and export options.
Versions
Use Versions from the ... menu to save named snapshots (e.g., “Draft for Review”). You can always restore a previous version — the current state is auto-backed up first.
Crowded days
Use Bands for presentation, but switch to Grid or Cards to validate crowded days before publishing. Layout applies to the whole schedule, not one day at a time.

FAQ
Where should I open DaySchedule from?
Open it from the shared team copy every time. If your team reaches it through SharePoint or Teams, use that shared copy on your computer instead of downloading a separate copy.
What folder do I pick?
Choose the folder where the schedule JSON files live. In most DaySchedule copies, that is app/data. If your team packaged it differently, choose the folder that already contains the saved schedules.
Can multiple people edit the same schedule at once?
The safest workflow is no. Have one person edit at a time per schedule, then hand off after the save indicator shows Saved.
What if the last editor stepped away?
Edit access uses a lease. If the last editor stops refreshing it, the lock expires automatically and the next person can click Edit. If the team lead needs in sooner, they can use Take Over after confirming the current editor is unavailable.
Where do I change the logo or look?
Use Customize in the toolbar. That is where title, unit logo, layout, colors, and audiences live.
Can different days use different layouts?
Not currently. Layout is schedule-wide, so switching to Bands, Grid, Cards, or Phases changes the full schedule.
Why did Main Track change when I changed Audience?
That is expected. A Primary audience automatically places the event in the main track. A Supporting or unassigned event stays out of the main track unless you turn on Main Track as an override.
What is the difference between Audience and Specific People?
Audience is the group, section, or shop the event belongs to. Specific People is only for named exceptions or callouts inside that audience.
What if I do not see someone else’s changes?
Confirm you opened the shared copy, not a personal one. Then return to the library and reopen the schedule so DaySchedule reloads the latest file.
What browser should I use?
Use Chrome or Edge for the full auto-save experience. Other browsers can still open the app, but saving falls back to manual Save to File.
